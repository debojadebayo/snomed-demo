You can toggle the left panel between a terminology server API lookup and an in-memory B-tree traversal. Even after data is loaded in memory, a classic B-tree style lookup still walks multiple pages/nodes to locate a term — each step is a comparison and a pointer chase. A perfect hash function (PHF) sidesteps that entirely: the concept ID is hashed into a slot index in O(1), and the value is fetched as a direct array lookup. SNOMED CT is a closed, versioned vocabulary (~360,000 concepts), so the PHF is compiled once per release and rebuilt when a new SNOMED release ships.
